## 模型发布/更新

### Kimi K3 发布：2.8T 参数开源模型，支持原生视觉与百万 token 上下文

月之暗面发布开源模型 Kimi K3。AI HOT 摘要显示，该模型拥有 2.8T 参数，采用 Kimi Delta Attention 和 Attention Residuals 架构，支持原生视觉能力与 100 万 token 上下文窗口。Kimi K3 的重点不只是参数规模，而是把超长上下文、多模态输入和开源路径放在同一个模型发布中，继续推动大模型基础能力向开发者侧下放。

📎 [Moonshot AI：Kimi Blog](https://www.kimi.com/blog/kimi-k3)

## 产品发布/更新

### Grok 推出 Automations：支持定时与邮件触发的自动任务

xAI 为 Grok 引入 Automations 功能。用户可以用自然语言描述任务，让 Grok 按一次、每日、工作日、每周、每月或每年等计划运行，也可以通过邮件触发并按发件人、收件人或主题过滤。每次执行都会形成完整对话并保存运行历史，结果可通过邮件或应用内通知返回。定时自动化面向所有用户开放，邮件触发需要 SuperGrok 订阅。Grok 正在从问答助手继续向个人自动化执行层扩展。

📎 [xAI：News](https://x.ai/news/grok-automations)

### Decoy 字体：用空间频率混淆让 AI 看不清输入文字

Decoy Font 是一款实验性 TTF 字体，通过在同一字符中叠加不同空间频率的图形，让 AI 近距离识别到“诱饵”字母，而人眼在远距离或眯眼时看到真实隐藏信息。它把对抗样式从图片提示扩展到字体层，提醒人们视觉模型、OCR 和多模态输入链路并不总能稳定理解人类真正想传达的文字。

📎 [Mixfont：Decoy Font](https://www.mixfont.com/experiments/decoy-font)

### ChatGPT 工作区支持文档、表格与幻灯片编辑

ChatGPT 官方账号演示了工作区中创建和编辑文档、电子表格与幻灯片的能力。这个方向把 ChatGPT 从对话式内容生成进一步推向可编辑办公对象，让用户在同一工作区内完成写作、数据整理和演示材料制作。对办公 AI 来说，关键竞争点正在从“生成一段文字”转向“持续维护可交付文件”。

📎 [X：ChatGPT (@ChatGPTapp)](https://x.com/ChatGPTapp/status/2077826846373380535)

### 天工短剧工作台发布“Agent 智能分镜 + 无限画布”双轨创作模式

天工短剧工作台推出双轨创作模式，用导演 Agent 自动解析剧本、规划站位和机位，并支持多视细节图生成，重点解决 AI 短剧里的角色变脸与站位漂移问题。工具内置影视级提示词模板、720 度全景图和 3D 导演台，面向更可控的短剧生产流程。AI HOT 摘要显示，已有三部作品上线 DramaWave 后 7 天实现百万美元级营收。

📎 [公众号：昆仑万维（天工）](https://mp.weixin.qq.com/s/WlGAeogkF_N5122nHA0TtQ)

### 秒哒 3.5 全球首发 iOS App，强化无代码开发与多端后端共享

百度智能云在 WAIC2026 发布秒哒 3.5，新增 iOS 打包能力，用户无需 Mac 或 Xcode 即可将应用打包为 IPA 文件或上架 App Store。新版还内置 SEO Agent，支持多应用共享同一后端数据库，并提供数据库多环境隔离与后端资源分级。秒哒累计服务超 3500 万用户，已创造 350 万个商业应用。无代码平台正在从网页应用生成，继续扩展到移动端发布和后端治理。

📎 [公众号：百度智能云（文心）](https://mp.weixin.qq.com/s/W3QACoSYgwK0TW4zG-KFsg)

### 面壁智能开源企业 AI 数字员工平台 StaffDeck

面壁智能联合多团队开源 StaffDeck，一个用于构建与管理数字员工的企业平台。它希望把专业知识、标准作业程序和决策规则沉淀为可持续工作、持续改进并保留组织知识的数字员工，而不是停留在聊天机器人形态。企业 Agent 的落点正在从“回答问题”走向“承接岗位流程”。

📎 [X：面壁智能 OpenBMB (@OpenBMB)](https://x.com/OpenBMB/status/2077741814799548451)

### MiniMax Code 2.0 桌面端焕新，金融模块即将上线

MiniMax Code 2.0 桌面端发布，基于 Pi Agent 框架重构底层架构，提升会话启动速度与长程复杂任务执行稳定性。新版本优化图表加载与文件预览框选编辑，并已与恒生金融数据库、企查查 MCP 打通，金融模块即将上线，支持多源数据实时检索与专业报告生成。编码 Agent 的桌面端正在叠加远程控制、浏览器操控和行业数据能力。

📎 [公众号：MiniMax（稀宇科技）](https://mp.weixin.qq.com/s/mQeBO0xC6Z1R0LqZX5TvNg)

### Claude Cowork 支持使用 Claude Fable 5 处理复杂异步工作

Anthropic 发布文章介绍如何在 Claude Cowork 中使用 Claude Fable 5。该模型面向长时间、多步骤的复杂异步工作，例如深度研究与尽职调查，需要用户手动选择，默认模型仍为 Claude Sonnet 5。这个产品路径说明，高端模型正在被包装进更明确的“托管任务”工作流中，而不是只作为一次性聊天模型出现。

📎 [Claude Blog](https://claude.com/blog/working-with-claude-fable-5-in-claude-cowork)

## 行业动态

### xAI 起诉 Grok 用户制作儿童性虐待内容，不再否认模型被滥用

xAI 首次对一名 Grok 用户提起诉讼，指控其利用模型制作儿童性虐待图像。AI HOT 摘要显示，xAI 此前一直否认 Grok 能生成此类内容，此次诉讼标志着其立场转向承认存在用户滥用风险，并把案件重点放在用户行为而非模型技术缺陷上。生成式 AI 平台的责任边界正在被真实案件不断重塑。

📎 [Ars Technica](https://arstechnica.com/tech-policy/2026/07/xai-cant-deny-grok-makes-csam-anymore-so-its-suing-users)

### 欧盟裁定 Google 必须向竞争对手开放 Android 和 Search

欧盟依据《数字市场法案》裁定 Google 必须向竞争对手开放 Android 和 Google Search 的关键部分，包括让第三方 AI 助手和搜索引擎获得更大访问权限。这两项决定可能削弱 Google 对移动操作系统和搜索入口的控制，也会影响 Gemini 等 AI 服务在平台层的分发优势。AI 竞争已经和平台互操作、数据访问和反垄断监管深度绑定。

📎 [The Verge](https://www.theverge.com/policy/966438/eu-google-android-ai-interoperability-search-data-dma)

### 台积电上调 2026 年资本支出预测至 600 至 640 亿美元

台积电在 2026 年第二季度财报说明会上将全年资本支出预测上调至 600 至 640 亿美元，此前预测接近 560 亿美元。董事长魏哲家表示，预计 2028 年量产的 A14 制程开发进展顺利，移动和高性能计算客户兴趣强烈，有望成为比 2nm 更大、更持久的工艺节点。AI 算力需求仍在继续推高先进制程和资本开支预期。

📎 [IT之家](https://www.ithome.com/0/977/544.htm)

### 世界人工智能合作组织协定签署仪式在上海举行

7 月 16 日，成立世界人工智能合作组织协定签署仪式在上海举行，中共中央政治局委员、外交部长王毅代表中国政府签署协定。该组织为独立政府间国际组织，总部设在中国上海，目标是促进人工智能国际合作与全球治理。AI HOT 摘要显示，哈萨克斯坦、老挝、巴基斯坦等 29 个国家代表签署协定成为创始成员国。

📎 [IT之家](https://www.ithome.com/0/977/793.htm)

### 54% 企业已遭遇 AI 智能体安全事件，多数仍共享凭证

VentureBeat 调查 107 家企业发现，54% 已遭遇 AI 智能体安全事件，其中 18% 确认事故，36% 险些酿成事故。仅 32% 的企业为每个智能体分配独立身份凭证，30% 会将高风险智能体隔离在沙箱中。安全工具仍主要依赖模型提供商原生方案，专用智能体安全产品渗透率很低。企业 Agent 的落地速度正在领先于身份、权限和隔离治理。

📎 [VentureBeat](https://venturebeat.com/ai/the-agent-security-gap-54-of-enterprises-have-already-had-an-ai-agent-incident-and-most-still-let-agents-share-credentials)

### Google DeepMind 与 Isomorphic Labs 公布生物弹性联合方案

Google DeepMind 与 Isomorphic Labs 公布生物弹性方案，围绕预防、检测和响应三类目标推进生物安全：防止模型滥用，使用 AlphaFold 和 IsoDDE 等技术加速疫苗与药物设计，并借助 AlphaEvolve 优化病原体监测。过去 12 个月，团队已与 15 个以上政府及生物安全组织建立合作，并探索将 SynthID 水印技术用于 DNA 合成筛查。

📎 [Google DeepMind Blog](https://deepmind.google/blog/our-approach-to-bioresilience)

## 论文研究

### HYPIC：小红书联合北大、上交提出混合注意力大模型位置无关缓存系统

HYPIC 面向混合注意力大模型实现位置无关缓存，将首 token 延迟平均降低 3.25 倍。AI HOT 摘要显示，在 4 个生产级模型上，同等服务目标下可持续 QPS 提升 1.66 倍，任务质量与完全重算仅相差 1.71 分。随着长上下文与混合注意力模型进入生产，缓存能否跨位置复用正在成为推理系统优化的关键问题。

📎 [公众号：小红书技术（dots.llm）](https://mp.weixin.qq.com/s/RWveWvw9yBH6YQINBQ-XjA)

### Moonshot AI 发布 PerceptionBench：多模态模型视觉感知能力诊断基准

Moonshot AI 发布 PerceptionBench，一个从 40 多个现有基准中的真实失败案例归纳出的视觉感知基准，包含 10 项原子感知能力和 3000 道验证题。所有测试模型准确率均未超过 60%，且大量正确答案在重复提问时无法稳定复现，说明模型很多时候是在猜测而非真正感知。这个基准把多模态评测从综合问答拆回更基础的视觉感知断点。

📎 [Moonshot AI：Kimi Blog](https://www.kimi.com/blog/perception-bench)

### Apple 研究：极简自我蒸馏提升代码生成能力

Apple 机器学习研究团队提出 Embarrassingly Simple Self-Distillation，让大语言模型仅使用自身原始输出提升代码生成能力，不需要验证器、教师模型或额外数据。AI HOT 摘要显示，该方法在多个代码生成基准上取得显著改进，并且计算成本较低。模型自我改进不一定只能依赖复杂外部反馈，简单蒸馏流程也可能带来可见收益。

📎 [Apple Machine Learning Research](https://machinelearning.apple.com/research/simple-self-distillation)

### Apple 用混合文本与 ID 嵌入实现增量视频搜索个性化排序

Apple 提出一种混合文本与 ID 嵌入方法，用于增量视频搜索中每次按键后的高质量排序。该方法在用户意图不明确时仍能提升个性化排序效果，例如只输入 1 到 3 个字符的早期查询阶段。对视频搜索和内容平台来说，个性化排序不只发生在完整查询之后，也需要在输入过程中的模糊意图阶段尽早介入。

📎 [Apple Machine Learning Research](https://machinelearning.apple.com/research/personalizing-incremental-video-search)

## 技巧与观点

### 企业 AI 智能体评估存在“现实对齐”缺口

一项针对 157 家企业的调查显示，50% 的组织在过去一年曾部署通过内部评估、但上线后导致客户故障的 AI 智能体或大语言模型功能。只有 5% 的企业完全信任自动化评估，29% 认为评估与现实结果对齐不佳是最大局限。与此同时，66% 的企业已经允许或计划在 12 个月内让低风险智能体全自动部署。企业现在的问题不是没有评测，而是评测结果与真实生产环境脱节。

📎 [VentureBeat](https://venturebeat.com/ai/the-agent-evaluation-gap-enterprise-ai-organizations-have-a-reality-alignment-problem-not-a-coverage-problem-and-most-are-shipping-to-production-anyway)

### Patter SDK 教程：构建餐厅预订电话智能体

Patter SDK 发布教程，演示如何构建餐厅预订场景的语音智能体工作流。流程支持动态调用变量、注册工具、输出护栏、延迟与成本指标追踪、回归式评估检查，以及在无需实时电话凭证的情况下运行脚本化通话模拟。这个案例把语音 Agent 的核心工程要素放在一条管线里：工具调用、安全检查、通话模拟和评估必须一起设计。

📎 [MarkTechPost](https://www.marktechpost.com/2026/07/16/patter-sdk-guide-to-building-a-restaurant-booking-phone-agent-with-dynamic-variables-guardrails-latency-dashboards-and-eval-checks)

### xAI 开源 Grok CLI 代码库中发现 Mermaid 转 Unicode 框图工具

Simon Willison 在 xAI 开源的 Grok CLI 编码智能体代码库中发现一个用 Rust 编写的 Mermaid 图表示例终端渲染器，位于 `xai-grok-markdown/src/mermaid.rs`。开发者随后用 Claude Code for web 将其编译为 WebAssembly，让工具可在浏览器中运行。这个小工具展示了开源 Agent 代码库中的周边能力也可能被快速拆出、移植和复用。

📎 [Simon Willison 博客](https://simonwillison.net/2026/Jul/16/grok-mermaid)

### 远程操控 Agent 干活方案：Codex 主力 + UU 远程兜底

作者分享远程使用 Agent 的组合方案：以 Codex 远程控制功能作为主力，通过 ChatGPT App 连接家中 24 小时开机的 Mac Mini，同步开发任务、规则和 Agent 记忆；遇到扫码登录、图形界面操作等 Codex 难以处理的场景时，再用网易 UU 远程在手机上直接操控电脑桌面。这个方案的价值在于承认 Agent 工作流仍会碰到登录、图形界面和人工确认这些现实边界。

📎 [公众号：数字生命卡兹克](https://mp.weixin.qq.com/s/x2JlkhOlGNVhL8oe7AVk9Q)

### Anthropic 用 Claude Code 大规模迁移代码：Bun 百万行 Zig 转 Rust

Anthropic 工程师分享用 Claude Code 在两周内将 Bun 的百万行 Zig 代码迁移至 Rust，现有测试全部通过，合并后出现的 19 个回归问题也已修复。另一个案例是周末将 Python 代码库迁移至 16.5 万行 TypeScript。迁移消耗约 16.5 万美元 API 成本，但编译时间从 8 分钟降至 2 秒，二进制启动快 6 倍。AI 代码迁移正在从局部重写迈向大型工程级案例。

📎 [Claude Blog](https://claude.com/blog/ai-code-migration)

### 千问 App 联合武汉发布举办 AI 求职实战课

千问 App 与武汉发布在武汉举办 AI 求职实战课，现场演示简历诊断、PPT 制作和表格分析。产品经理提出“给全材料、说明目标、定义标准、划定边界、索要可编辑文件”五步法，并展示将 486 行杂乱销售数据浓缩为一页结论 PPT 的“建、理、算、析、呈”方法论。这类课程说明通用 AI 助手正在被包装成面向具体场景的职场训练工具。

📎 [公众号：千问APP（阿里）](https://mp.weixin.qq.com/s/dCk6IXbFyOSSc1JxcWsglA)

### Google 提出模块化提示词转译方案，构建可扩展 AI 智能体

Google 提出将系统提示词模块化为可复用的“技能文件”，通过转译器做静态验证，并在构建时捕获缺失依赖，从而缓解单体提示词带来的扩展瓶颈和运行时错误。该方法可把提示词生成接入 CI/CD 流水线，防止代码漂移，并允许智能体通过标准 pull request 自主更新逻辑。提示词工程正在越来越像软件工程：模块化、验证、构建和审查缺一不可。

📎 [Google Developers Blog](https://developers.googleblog.com/building-scalable-ai-agents-with-modular-prompt-transpilation)

### OpenAI 强调青少年应获得安全 AI 访问权

OpenAI 发文主张青少年应获得安全的 AI 访问权，并介绍面向 teen 用户的保护措施。AI HOT 摘要显示，每周有近 9 成 teen 用户使用 ChatGPT 进行学习、信息获取或技能提升。OpenAI 已推出 Study Mode、年龄预测、家长控制，以及针对暴力、自残和色情角色扮演等内容的安全护栏。围绕青少年使用 AI 的讨论，正在从“是否允许使用”转向“如何以默认安全的方式使用”。

📎 [OpenAI](https://openai.com/index/why-teens-deserve-access-safe-ai)
